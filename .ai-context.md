# MobileCube AI Context

## @TechStack

- **Languages**: JavaScript (Vanilla/ES6), HTML, CSS
- **Runtime/Build**: Docker Compose mapped to Python 3 for Hot-Reloading Dev Server (`Dockerfile.dev`)
- **Ecosystem**: Twine (Story Format, based on SugarCube 2)

## @FileStructure

```text
mobile-cube/
├── .vscode-templates/
├── dist/                # Output directory for format.js
├── src/                 # Source code
│   ├── css/             # Core stylesheets
│   ├── js/              # Engine, state, macros, and UI logic
│   └── index.html       # Base HTML wrapper
├── test-story/          # Likely contains a Twine twee/html project for testing
├── ROADMAP.md
├── Taskfile.yml         # Task runner for VSCode deploy
├── build.js             # Node build script
├── build.py             # Python build script alternative
└── package.json         # Node metadata and build command
```

## @Contract

- **Build System**: Start the hot-reloading development server by running `docker compose up dev` inside the `mobile-cube` directory. This handles building the `format.js` engine and continuously recompiling the `test-story` HTML over `localhost:8000`. You can also execute single task builds via VS Code Tasks linked to Docker overrides.
- **Framework Target**: Generates a Twine `storyFormat` JSON payload wrapped in a JS function call `window.storyFormat(...)`.
- **Primary Design Goal**: Mobile-first, one-handed navigation `[needs-verification on responsive breakpoints]`.
